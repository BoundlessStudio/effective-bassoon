# OpenAI Realtime Music Co-Pilot

A split-stack demo that pairs a Vue 3 (Vite) front end with a Node/Express backend to experiment with OpenAI's Realtime API for voice-enabled music workflows. Upload a Suno.AI instrumental or full mix and explore automated analysis, lyric ideation, lyric refinements, and even a playful sung hook generated by the assistant.

## Stack

- **Client:** Vue 3 + Vite with Axios for API calls.
- **Server:** Node.js + Express, Multer for uploads, WebSocket bridge to OpenAI Realtime API.

## Features

- Upload MP3/WAV files exported from Suno.AI or other DAWs.
- Choose between music analysis, production coaching, lyric brainstorming, lyric polish, or an AI vocal hook.
- Backend relays the audio to the OpenAI Realtime model and returns text (and optional audio) responses.
- Frontend streams responses to the UI and renders downloadable audio when available.

## Getting Started

### 1. Install dependencies

```bash
cd server && npm install
cd ../client && npm install
```

### 2. Configure environment

Copy the `.env.example` in the `server` folder and fill in your OpenAI credentials.

```bash
cd server
cp .env.example .env
# edit .env to add your OPENAI_API_KEY and optional overrides
```

> **Note:** Realtime voice support currently requires access to `gpt-4o-realtime-preview` (or the model specified in `OPENAI_REALTIME_MODEL`). Ensure your account is enabled for realtime WebSocket access.

### 3. Run the development stack

Start the backend (port 3001 by default) and the frontend (Vite dev server on port 5173):

```bash
# Terminal A
cd server
npm run dev

# Terminal B
cd client
npm run dev
```

Navigate to [http://localhost:5173](http://localhost:5173) to use the app. The Vite dev server proxies `/api` requests to the Express backend.

### 4. Deploy / production build

- Build the client: `cd client && npm run build`
- Host the generated `dist` folder on any static host.
- Deploy the backend to your preferred Node environment and expose the `/api/analyze` route.

## How it works

1. The Vue UI lets you select the realtime mode and optionally paste lyric context.
2. Upon submission, the file is uploaded to the Express server.
3. The server streams the audio buffer into an OpenAI Realtime WebSocket session and issues a `response.create` event with tailored instructions.
4. Once the realtime response completes, the server returns structured JSON (text + optional base64 audio) back to the client.

## Caveats & Notes

- The Realtime API is still evolving—expect payload shape changes. Adjust parsing in `server/src/realtime.js` as needed.
- Singing (`sing-demo`) requests for an audio hook expect audio output support. The response includes `audioBase64` when available.
- Error handling is kept simple for clarity. Inspect server logs for detailed troubleshooting.
- Always comply with OpenAI usage policies when uploading and processing audio.
